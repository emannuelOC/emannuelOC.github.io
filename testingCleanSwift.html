Testes Unitários + Clean Swift

Você já ouviu do famoso problema do Massive View Controller (MVC)? Se não, mais cedo ou mais tarde você vai encontrar esse termo por aí. Se você está desenvolvendo para iOS há algum tempo, com certeza já viveu exemplos desse fenômeno. Sabe quando uma mesma classe (no caso, um ViewController) é responsável pelo layout das views, update dos outlets, cuidar das actions, funcionar como Data Source e Delegate para uma tableView ou collectionView, criar e apresentar meia dúzia de outros ViewControllers, ser delegate do ImagePickerController, etc.? Bom, nesse caso, você tem uma classe gigante ("Massive") com muitas responsabilidades e bem difícil de testar.

Pois bem, eu disse tudo isso só pra explicar o que o Clean Swift. Existem muitas abordagens para lidar com o problema do Massive View Controller (MVP, MVVM, VIPER, etc.) e um deles é o Clean Swift, que é baseado na Clean Architecture do Uncle Bob.

Essa arquitetura tem os seus pontos positivos e negativos. Há quem a adore, quem a odeie e eu não vou comentar sobre isso agora (talvez no futuro?)

De qualquer forma, o Rodrigo Soldi e eu usamos essa arquitetura em alguns projetos ao longo desse ano e na semana passada decidimos criar um repo para exemplificar o uso de Unit Tests pra essa arquitetura

Lá tem um README que explica basicamente os motivos, as dificuldades e os truques em cada classe de teste. Que fique bem claro: não é uma aula, um tutorial, nada disso. É simplemente um exemplo bem simples sobre como testar o seu código quando se usa Clean Swift.
